Azure Trusted Signing

formerly Azure Artifact / Code Signing

Cloud-native code & artifact signing

Sign your software at scale — without managing certificates.

Azure Trusted Signing is a fully managed, Microsoft-operated signing service. You authenticate with Microsoft Entra ID, and Microsoft handles certificate issuance, secure key storage in FIPS 140-2 Level 3 HSMs, rotation, and revocation — so you never touch a private key.

What it is & how it works

Signatures are short-lived: a certificate is minted just for your signing request and is timestamped, so the signature stays valid long after the certificate expires. Pick a stage to explore the flow.
